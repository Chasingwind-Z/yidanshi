# 一箪食 · Roadmap

> 2026-07-18 定稿。依据：产品研究 agent 的竞品调研（下厨房/懒饭/薄荷健康/Mealie/Tandoor/日食记/电子菜单小程序）
> + 独居做饭人群需求分析 + 用户（箪主）真实诉求：独居、周末做饭、控饮食、怕做完忘、不想重样、未来给亲友点菜。

## 立身之本（已验证的定位）

「做过就忘、菜谱难沉淀成自己的」是真实存在且**无人做好**的需求（豆瓣/知乎/V2EX 多处讨论，
「简单吃点」评论区 146 赞求个人菜谱录入）。一箪食 = 私人食单手账，这条主线不动摇。

竞品启示速记：懒饭赢在「用量精确不坑人」；薄荷赢在热量数据但记录摩擦大；Mealie 的购物清单
和 Tandoor 的份量换算是自托管圈公认最有用的两个功能；日食记证明「一人食的氛围感」本身就是产品；
决策类小程序证明「今天吃什么」是最高频痛点。

## 留存生死线（所有迭代的第一约束）

饮食记录类产品 3 周内流失过半，主因：记录太慢、断签羞耻。因此：
**记一餐必须 30 秒内完成；永不做连续打卡机制；补记要体面（已有「昨天（补记）」快捷键）。**

---

## 近期（下 1–2 轮，低成本高命中）

- [ ] **翻牌子加条件**：随机时可选「最近 7 天没做过」「≤30 分钟」（需菜谱加耗时字段），一次解决决策疲劳 + 不重样
- [ ] **月度食单回忆卡**：食历自动汇成一张可分享的月度小结图（本月 X 道菜 / 新解锁 Y 道 / 最高分），摆盘图直接复用——留存抓手 + 晒图素材
- [ ] **实测量回填提示**：记一餐保存后若该菜谱有"适量"类模糊量，轻提示「这次放了多少？回填一下下次照做」——个人版的懒饭精确量
- [ ] **热量周视图**：食历顶部按周合计 kcal（菜谱级热量已就位，零额外记录成本）
- [ ] 教程卡导出为长图（分享/打印），复用现有卡片版式

## 中期（需求已被竞品验证，需要新模块）

- [ ] **点菜链接**：生成只读食单页（含翻牌子），亲友手机打开即可点菜，点单结果进食历待办——
      不上小程序也能做：局域网直接开；外网走 Cloudflare Tunnel/Tailscale Funnel 免费隧道。
      「翻牌子/朱批/圣旨」宫廷叙事一脉相承，是被验证的情侣传播爆点
- [ ] **份量换算 + 本周买菜清单**：菜谱按份数换算食材量；勾选本周想做的菜合并生成购物清单（Tandoor/Mealie 最受欢迎功能）
- [ ] **食材库存轻管理**：记录冰箱里有什么，翻牌子优先推荐「现有食材能做的」（竞品评论区高赞需求）
- [ ] 菜谱耗时/难度字段（AI 整理时顺带估）

## 远期

- [ ] **微信小程序版**：触发条件 = 点菜链接被真实使用后（微信内传播效率最高）；评估已备好 docs/miniprogram-eval.md
- [ ] 营养轻周报（蛋白/蔬菜多样性温和提示，「了解自己」而非「审判自己」）
- [ ] 社媒链接直接导入（现阶段"复制文案粘贴"路线正确，等合规通道）
- [ ] 多人食单（合租/家庭各自记录共用一份菜单）

## 宣传点（当恢复推广时，只讲三个）

1. **「随口说说做法，AI 给你出一张印刷级插画菜谱卡」**——视觉反差感，全场唯一的画面，首图
2. **「把做过的每顿饭，攒成一本自己的菜谱书」**——"三个月攒下 28 道菜"式数字叙事
3. **「今天吃什么？让 TA 翻牌子」**——最大公约数痛点 + 宫廷梗记忆点 + 预告点菜玩法

（「免费开源、数据是自己的本地文件」不作主卖点，放正文当信任状，钩自托管圈层二次传播。）
