# 竞品吸收清单 + 定位收紧（2026-07-21）

> 背景：zzf 发现家庭点菜方向"越做越像饭店扫码点餐"，要求认真调研同类产品，
> 吸收功能/UI/理念优点，同时把一箪食自己的定位做深。本文是调研结论 + 吸收决策。
> 策略同步明确：**以个人使用为主；仓库保持公开但小红书不放 GitHub 链接，只晒使用效果，
> 呼声大再开源放配置教程。**

## 一、赛道地图（2026-07 调研）

**家庭电子菜单/点菜小程序**是一条已成型的赛道，头部产品：

| 产品 | 定位 | 亮点 | 短板 |
|---|---|---|---|
| **食遇日记**（测评 58.5/60） | "和家人朋友一起制作人生菜单" | 自动精美菜卡、共享厨房、心愿清单、协作点餐、打卡日历、电子菜单导出、点赞/庆祝微动效、全免费无广告 | （测评中几乎无差评） |

小红书一手信号：
- **"我给对象/家人做了个点菜小程序"是已验证爆款格式**：一条视频 2184 赞/2207 转发；
  "给对象全家用的点菜 app 后后后续"在连载；"用 Gemini3 搭建家庭菜谱" 915 赞/773 收藏。
  → 佐证"先晒使用效果再看呼声"的路线；**叙事主角是"我为家人做的"，不是产品功能**。
- 需求侧真实存在："想记录自己做的菜"（70 评论）、"有没有替代厨神与吃货的小程序"。

## 二、定位收紧（一句话）

竞品全是**电子菜单**——菜单是拿给别人看/点的。
一箪食是**自己做菜的成长档案**——插画教程卡、两口径营养、实测量回填越做越准、
食历时间线；点菜只是家人参与这份档案的一扇小门。

要做大的独有优势（竞品全都没有）：
1. **AB 风插画教程卡**（贴纸厚涂 + 锚点体系）——独一份的视觉资产；
2. **营养两口径诚实计算**（每餐/整锅分离，来源三级标注）；
3. **实测量回填**——菜谱越做越精确，这是"档案"而非"菜单"的核心证据；
4. **数据全归自己**（自托管 + 文件库）；
5. 翻牌子**条件**随机（7 天不重样/时长/难度/冰箱食材），比竞品的裸转盘强。

## 三、吸收决策

### 现在就吸收（进 P1 家庭点菜）
- **协作点餐的"温度"措辞**（食遇日记的理念"厨房到餐桌是爱的最短距离"）：
  全程叫「想吃 / 点菜 / 传旨」，禁用 订单/下单/购物车/结算。
- **每道菜可带备注**（zzf 明确要的：少放辣、多放醋、不要香菜）。
- **角色极简**（御膳房反面教训：权限图标老人看不懂）：只有"主人/客人"两种，
  客人无需任何设置，点开分享卡片即达点菜页。
- **低学习成本**（今天吃什么的"先轻后重"）：客人侧一屏完成 选菜→备注→留名→传出去。

### 排入 backlog（P3 候选，各一句话）
- **电子菜单导出**（食遇日记/今天吃什么都有；且是小红书晒图素材机器）——
  把食单渲染成一张可保存的"纸上食单"长图/海报。已有教程卡导出底子，扩展成本低。
- **完成记录时的庆祝微动效**（食遇日记）——记一餐保存成功时轻庆祝，强化仪式感。
- **食材保质期提醒**（御膳房）——pantry.json 已有库存底子，加个购买日期+提醒即可。
- **放大模式/适老**（今天吃什么）——若爸妈辈真实用起来再做。

### 明确不吸收
- 付费/积分/会员体系（厨神与吃货的反面教训；自托管无此需求）。
- 社区/广场/他人菜谱浏览（是"菜谱平台"的事，一箪食是私人档案）。
- 体重管理/奶茶打卡等泛健康记录（今天吃什么有——对一箪食是范围蔓延，
  营养目标已覆盖核心需求）。
- 任何饭店点餐痕迹：桌号/催菜/后厨状态/支付。

## 三点五、二轮调研增量（2026-07-22，R20 调研 agent，一手小红书 12 篇+源码级拆解）

**验证了的判断**
- 「每日推荐给理由 + 健康约束」是**全行业空白**：竞品（今天吃什么/御膳房/好大一颗菜）
  全是裸随机转盘，无一家给推荐理由——R20-a 每日荐的差异化成立。
- 「成长档案」心智真实存在：食遇日记官方话术已在用"味觉成长史"，说明该定位被市场验证，
  也说明要靠插画/营养/实测量这些硬资产拉开，光靠叙事不够。

**长图导出的设计输入（已按第五节 5 条建议转译进 R20-b 实现）**
- 晒图主流形态 = 宽 1179~1440px 的 1:2~1:6 竖长图；手绘/手帐质感碾压 App 截图
  （57899 赞断层第一是纯手绘菜谱卡）；分页阈值 18/12 是竞品验证的工程参数。
- 竞品导出图一律无二维码/水印——长图是纯"晒"介质，落款只留品牌记忆点。
- 评论区声量第一是**「求克隆码/求模板」**：长图若火，用「空白食单模板图」接流量
  （发帖时预备，评论区直接回图），呼应"呼声大再开源"。
- 标题公式=关系叙事（"给男朋友做的…"/"究竟是谁还没有 XX"），主角是关系不是工具。
- 抠图是评论区公认痛点（"抠图怎么弄"高频）——rembg 自动抠图是现成的传播卖点。

**新进 backlog 的（各一句话）**
- **划掉重抽**（V2EX 洞察："问题是筛不是选"）：翻牌子抽中不满意→划掉→池内重抽，当日不再出现。
- **备餐/预点日期**（竞品评论区最高赞需求）：点菜时可选"哪天吃、早午晚"——家庭点菜 P2 候选。
- **食历月度长图**：月结卡（monthcard.py）已是现成的第二晒图体裁，发帖素材+1。
- **「点赞过 100 就上线」互动钩子**：发小红书时把 roadmap 做成评论区互动（BaobaoE 验证 424 赞）。

## 四、开源与传播策略（2026-07-21 更新）

- **仓库保持公开，但不引流**：小红书发帖**不放 GitHub 链接**；继续自用打磨。
- **小红书先晒使用效果**：素材角度按已验证格式——"我给自己/家人做了个点菜小程序"
  个人叙事 + 插画教程卡/纸上食单视觉差异点。发帖时机 = 家庭点菜真实用起来之后。
- **呼声大再放链接**：届时公开推荐仓库 + 补配置教程（docs/deploy.md 已是现成底子）。

### 宣发图模式（2026-07-22 定调，待产品打磨够再执行）
**不用**科技感设备样机（shots.so/渐变悬浮手机壳那套——适合效率工具卖科技圈；
本赛道手绘手帐 5.8 万赞 vs App 截图型天花板 ~2400，差 20 倍；且纸质感是自有资产，
套冷灰样机等于自废武功）。**用「实物感」三件套**：
1. 首图 = 关系叙事标题（朱砂大字，如「一个人住，我给来家吃饭的朋友做了个点菜小程序」）
   + 教程卡/长图"斜放在木桌上的实拍感"拼贴；
2. 内容页 = 产品自产图直接上（教程卡/纸上食单长图天生 1080 竖图小红书体质）
   + 一张「拍照→抠图前后对比」（评论区公认痛点钩子）；
3. 过程视频 = 真机录屏 15s（拍→抠→菜卡蹦出 / 朋友传旨→微信弹提醒→盖章）。
配套：评论区备「空白食单模板图」接"求模板"、「点赞过 100 加 XX」互动钩子。
**待做工具**：首图生成器（PIL 管线同 monthcard：宣纸底+朱砂标题+卡片斜放投影，
一条命令出 3:4 封面）——等 zzf 觉得产品够了再开工。
