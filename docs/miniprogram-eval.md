# 微信小程序移植评估（G7）

结论：**Web 版验证出真实留存后再动手**；移植成本可控，最大的坑是抠图通道，其次是个人主体的能力边界。

## 成本

| 项目 | 说明 |
|---|---|
| 小程序注册 | 个人主体免费，需实名；名称不能带「菜谱」等需资质词，「一箪食」这类品牌名可用 |
| 云开发 | 基础套餐约 ¥19.9/月（含云函数、云数据库、云存储、CDN），个人主体可开通，免备案 |
| 审核 | 个人主体类目选「生活服务-其他」；纯记录工具类通常可过，含 UGC 分享功能会被要求内容安全接口 |

## 抠图通道（关键难点）

云函数跑不动 rembg（内存/包体积/冷启动都不合适），三个可行方案：

1. **Mac 出服务**（推荐起步）：本机 18100 端口的 `/api/cutout` 经 frp/Cloudflare Tunnel 暴露 HTTPS，云函数转发调用。零新增成本，依赖家里 Mac 在线。
2. **付费 API**：腾讯云「图像分割」/阿里云「通用抠图」按次计费（约 ¥0.01–0.1/次），个人用量每月几块钱。稳定，不依赖 Mac。
3. **端侧兜底**：引导用户 iPhone 相册长按抠图后上传透明 PNG，零成本，Android 覆盖差。

## 数据 / UI 映射

- 文件数据库 → 云数据库两个集合（recipes / meals），字段与现有 schema 一一对应；Markdown 正文整体存字符串字段即可，解析逻辑照搬 `server/storage.py`
- React 组件 → 小程序 WXML/WXSS 或 Taro（React 语法直接复用，推荐 Taro 降低移植量）；hash 路由 → 小程序页面栈
- 深色主题 CSS 变量原样搬；PWA 加主屏 → 微信「添加到我的小程序」
- 「暗号」邀请制（参考 Taste）：云函数校验一个白名单即可，避免开放注册带来的内容安全义务

## 触发条件（建议）

Web 版连续记录 ≥ 4 周、或小红书发帖后有明确「想要微信版」的需求信号，再启动移植。
