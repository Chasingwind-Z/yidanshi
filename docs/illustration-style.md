> 适用范围：菜谱「插画教程卡」的两类图 —— 左栏**食材图标**（每食材一张方图）、右栏**步骤插图**（每步一张横图）。

> 本文档只定义风格与 prompt，不涉及生图 API 选型与调用代码。

## 1. 风格定位

**一句话**：黑底上的「贴纸式手绘卡通」食物插画 —— 粗深棕描边 + 半厚涂（gouache）上色 + 暖黄顶光，主体像贴纸一样悬浮在纯黑背景上，无环境场景。

**三条核心特征**（来自对 AB 卡的逐项观察）：

1. **强轮廓贴纸感**：所有物体带中粗、深棕近黑的均匀描边（线宽约为画面宽度的 1.5%–2%，略带手绘抖动），剪影完整、彼此不粘连，缩到 72px 仍能一眼认出。
2. **半厚涂暖光上色**：不是扁平色块，也不是写实渲染 —— gouache 质感的两阶明暗（固有色 + 暗部）加一点高光点，柔和暖黄顶光，整体色温偏暖、饱和度偏高（翠绿芦笋、粉红牛排、乳白蒜头都比实物更「鲜」半档），在黑底上自然发亮。
3. **无场景道具拼贴**：不画厨房、桌面、墙面。步骤叙事靠 2–4 个卡通化道具横向拼贴 + 一套固定符号语言完成：浅色短箭头表示状态变化、秒表图标表示计时、锅底橙色火苗表示加热、锅上灰白蒸汽表示出锅前、调料瓶用中文短标签点名调料。黑底本身就是留白。

### 为什么是这一路（流派对比结论）

| 流派 | 例子 | 黑底 + 小尺寸表现 |
|---|---|---|
| 扁平矢量图标风 | 常见 UI 图标库 | 清晰但单薄，像 emoji，无食欲、无手作感 |
| 水彩手绘风 | Salt Fat Acid Heat 式插画 | 边缘发虚、对比低，黑底上发灰糊成一团 |
| 日式料理漫画/图鉴线稿风 | 深夜食堂、《料理图鉴》 | 线条叙事强，但线稿为主的画面在黑底吃亏 |
| **贴纸式厚涂卡通风（选定）** | AB 教程卡、die-cut sticker 风 | 粗描边保轮廓、高饱和保识别，黑底小尺寸最清晰 |

## 2. 风格一致性锚点（所有图共用的固定前缀）

**规则：以下前缀逐字不变地放在每条 prompt 开头，任何图只允许改前缀之后的内容描述。** 改了前缀 = 换了画风，禁止。

**STYLE_ANCHOR（中文版，用于支持中文的生图 API：即梦/Seedream、混元、GPT-image 等）**

```

```

sticker-style hand-drawn cartoon food illustration, thick dark-brown outlines with uniform weight and slight hand-drawn wobble, semi-impasto gouache coloring (base color + one shadow step + small specular highlights), soft warm yellow top lighting, saturated warm palette, on a pure black background (#000000), subject floats like a die-cut sticker with a clean complete silhouette, no environment, no tabletop, no cast shadow

## 3. 食材图标 prompt 模板

**用途**：教程卡左栏，一个食材一张图，UI 在图下方渲染食材名（名称**不**烧进图里）。

### 模板

```
{STYLE_ANCHOR}
画面内容：{食材描述}。
构图：单一食材居中，主体占画面 70%–80%，四周留出黑底边距，整体收在画面内切圆内（适配圆形裁切）；正面或 3/4 略俯视角度。
{NEGATIVE_COMMON}
另外不要：多种食材混在一张图、切配好的成品菜、餐具堆叠。
```

### {食材描述} 的写法规则

- **画「备菜的量」而不是孤立标本**：芦笋画一小捆 6–8 根，蒜画一头带 2–3 瓣散蒜，葱画 2–3 根 —— 有「准备好了」的生活感。
- **点名关键识别特征**：颜色、纹理、形态各给 1 个具体词（如牛排的「白色大理石脂肪纹」、蒜的「浅棕根须」），防止模型画成泛型蔬菜/泛型肉。
- **承托物只在必要时用**：生肉、液体、粉末等摊不住的食材，允许放在**一只奶白色浅盘**上（AB 卡牛排的画法）；其余食材直接悬浮，不加盘子。
- 禁止在描述里写任何要出现的文字。

## 4. 步骤插图 prompt 模板

**用途**：教程卡右栏，一步一张横图（4:3），左侧是 UI 渲染的步骤文字，插图只负责「看图会做」。

### 模板

```
{STYLE_ANCHOR}
画面内容：{道具与动作描述，按时间顺序从左到右排列}。
构图：横幅画面，2–4 个主体横向拼贴排列（成组的调料瓶算一个主体），阅读顺序从左到右，主体之间留出黑底间隔，黑底留白不少于画面 25%；整体柔和暖光，无地面线。
{NEGATIVE_COMMON}
另外不要：完整厨房场景、灶台台面、多余装饰道具、超过一个箭头、步骤编号数字。
```

### 场景与符号规则（{道具与动作描述} 怎么写）

| 要表达的 | 画法 |
|---|---|
| 厨具道具 | 与食材同一套描边+上色卡通化：木纹砧板（暖棕）、不锈钢锅（灰蓝底+暖高光）、平底锅（深灰 `#2a2a2a` 起步，**禁纯黑**，锅沿加亮边）、厨房纸卷、菜刀等；道具是配角，只画该步骤用到的 |
| 正在加热 | 锅底画 2–3 簇橙黄色小火苗 |
| 快出锅 / 热气 | 锅上方画 2–3 缕浅灰白蒸汽弧线 |
| 状态变化（切好→下锅、锅 A→锅 B） | 两个主体之间画**一个**米白色短粗箭头，指向右；一张图最多一个箭头 |
| 计时 | 画一只卡通秒表/计时器图标（银灰表壳、表盘朝前）；**时间文字由 UI 叠加在图旁，不让模型写数字**（AI 数字易崩） |
| 具体调料 | 调料瓶/研磨瓶造型区分（酱油玻璃瓶、蚝油深瓶、研磨瓶），瓶身可带白底标签写 **2–4 个中文大字**（如「生抽」「蚝油」）；生成后必须人工检查字形，崩字则回退：让标签留白，由 UI/后处理贴字 |
| 切、腌、拌等动作 | 用「动作完成态 + 工具在场」表达：切好的条状肉 + 菜刀在旁，不画手 |

**密度硬上限**：主体 ≤4、箭头 ≤1、计时器 ≤1。超了就拆步骤，不要挤进一张图。

## 6. 尺寸与深色 UI 适配

### 尺寸

| 图类 | 生成尺寸 | 展示尺寸 | 比例 |
|---|---|---|---|
| 食材图标 | 1024×1024（API 最小档也可 512×512） | 512×512，UI 内实际显示约 64–96px | 1:1 |
| 步骤插图 | 1024×768 | 768×576 | 4:3 |

生成取高一档、缩小交付：缩小能收紧描边毛边，贴纸感更干净。

### 深色 UI 适配注意点

应用深色主题实测底色：卡片/容器用 `#000`、`#0d0d0d`、`#111214`、`#10131a`、`#1a1c20`。

1. **黑底直出，不走透明抠图**：prompt 固定要求纯黑 `#000000` 背景直出。白底生成再抠图会在描边外留下白色/半透明毛边（matting fringe），黑底上非常显眼。
2. **图片容器底色统一为 `#000`**：插图背景是 `#000`，若容器用 `#111214` 等近黑色会露出一圈色差方块；放图的容器（含圆角裁切区）一律铺 `#000`，或给图片加与容器同色的底。
3. **主体内禁大面积纯黑**：铸铁锅、酱油瓶等深色道具最深只到 `#2a2a2a` 左右并加暖色亮边/高光，否则与背景融成剪影缺口（模板里已写进锅具画法）。
4. **小尺寸自检**：每张食材图标缩到 72px 检查一遍 —— 剪影可辨、描边不糊、不与相邻图标混淆；不过关就简化描述（减根数、减瓣数）重生成。
5. **文字策略**：一切必读文字（食材名、步骤号、时长）由 UI 渲染；图内文字只允许调料瓶 2–4 字中文标签这一处，且逐张人工校验，崩字回退空白标签。
6. **同批一致性**：同一道菜的所有图尽量同一模型、同一批参数生成；换模型必须整菜重生成，禁止混批。

## 7. 实测结论（2026-07-18，doubao-seedream-5-0-pro，芦笋牛排全套 6 张）

- **风格命中率 6/6**，零废图：描边、厚涂、暖光、黑底悬浮全部符合锚点；72px 缩图自检通过。
- **模型自发给步骤图加了奶白色 die-cut 贴纸边**（负面清单里禁过但没拦住）。实际效果加分且三张步骤图内部一致，**决定保留为步骤图的正式风格特征**；食材图标无此边，两栏形成「直绘食材 / 贴纸步骤」的自然区分。负面清单中「白色贴纸描边」条目对该模型可忽略。
- 模型倾向省略调料瓶中文标签（步骤 1 未画生抽/蚝油瓶）→ 零崩字风险，符合「文字由 UI 渲染」的原则，不需干预。
- 耗时约 60 秒/张；成本 0.3 元/张（1440×1440 与 1728×1296 均落在 ≤236 万像素档），一道菜全套 ≈ 1.8 元。
