# 一箪食 · 路线图

按 Goal 推进，每个 Goal 是一次会话可完成的量，做完勾掉并更新「验收」结果。
排序即优先级；G1 抠图管线是核心价值，最先验证。

## G0 项目骨架

- [ ] 定技术栈：FastAPI（Python，后端 + 抠图管线）+ Vite React PWA（前端，深色主题）+ 文件数据库（无 SQL）
- [ ] 目录结构：`server/`、`web/`、`data/recipes/`（菜谱 Markdown+frontmatter）、`data/meals/`（吃饭记录 JSON）、`data/photos/`（原图/抠图/菜卡）
- [ ] 定义数据 schema：菜谱（名称、分类、食材、步骤、贴士、封面）/ 一餐记录（日期、菜谱引用、照片、评分、备注）
- [ ] 本地 dev 起服务，空菜单页可打开

**验收**：`README` 里的开发启动命令能一条跑起来，看到空菜单页。

## G1 抠图美化管线（核心价值，最先验证）

- [ ] rembg + BiRefNet 抠图函数：俯拍碗盘照 → 透明 PNG
- [ ] 菜卡合成：透明 PNG → 深色底 + 居中 + 柔和阴影 → 菜卡图（参考 Taste 的黑底悬浮盘效果）
- [ ] 兜底通道：直接上传已抠好的透明 PNG（iPhone 相册长按抠图导出）
- [ ] 用 3 张真实做饭照验证效果，结果截图存 `docs/`

**验收**：一条命令 / 一个接口输入原图，输出能看的菜卡图。

## G2 菜谱库 + 记一餐

- [ ] 菜谱 CRUD：页面新建/编辑，同时支持直接改 `data/recipes/*.md`（AI 助手录入通道）
- [ ] 记一餐流程：上传照片 → 自动抠图 → 选已有菜谱或新建 → 填日期/评分/备注 → 保存
- [ ] 吃饭时间线页：按日期倒序的记录流
- [ ] 菜谱详情页：封面菜卡 + 品味均分 + 做过几次 + 教程入口

**验收**：完整记录一顿真实的饭，并在时间线和菜谱页看到它。

## G3 菜单页（Taste 风格）

- [ ] 分类侧栏（一碗饭 / 一碗面 / 一碗汤 / 一碗菜，可自定义分类）
- [ ] 菜卡列表：抠图照 + 名称 + 品味评分 + 做过几次 + 查看做法
- [ ] 深色主题打磨（等宽数字、描边按钮等 Taste 质感细节）
- [ ] 「随便来一份」随机推荐按钮

**验收**：手机上打开菜单页，观感与 Taste 同级。

## G4 插画教程卡

- [ ] 卡片版式：顶部菜卡照 + 菜名（by zzf）+ 左栏食材图标 + 右栏编号步骤配插图 + 底部小贴士
- [ ] 插画生成管线：统一画风 prompt 模板（卡通食材/步骤图，深色底），批量出图可委派 Codex image_gen
- [ ] 教程录入流：社交媒体教程（链接/文字/截图）→ AI 整理成结构化菜谱 → 人工确认
- [ ] 无插画时的纯文字降级版式

**验收**：为 2 道常做的菜生成完整插画教程卡，照着能做出来。

## G5 部署自用

- [ ] launchd 常驻 Mac（参考本机既有服务的部署方式）
- [ ] 手机访问：局域网 / Tailscale，PWA 加主屏（图标、启动屏）
- [ ] 数据备份策略（data/ 定期打包或另行 git 私有仓库）

**验收**：手机主屏点开即用，连续记录一周真实晚饭。

## G6 开源打磨 + 小红书首发

- [ ] README 配效果截图 / demo GIF、示例数据（避免真实隐私数据入库）
- [ ] 一键部署（docker compose 或单脚本），降低他人上手门槛
- [ ] 小红书首发帖：「我把自己做过的每顿饭做成了一份菜单」+ 制作过程素材
- [ ] 竞品评论区（简单吃点 / Taste）求「个人菜谱录入」的用户是天然种子用户

**验收**：陌生人按 README 能 30 分钟内跑起来；首发帖发布。

## G7（后续）微信小程序移植评估

- [ ] 评估个人主体小程序 + 云开发成本，抠图走 Mac 服务或付费 API
- [ ] Web 版验证的数据结构 / UI 直接映射到小程序
